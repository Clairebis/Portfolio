TrackTap

International Summer School


Task: Implement an application prototype using React Native.

Following a brainstorming session, the concept we worked with for this project was a music app aimed at allowing users to send song requests to a DJ / party host.

The problems we identified were i. Users have to fight to get the DJ's attention, ii. Users have to compete with the music and other partygoers to have their request heard, and iii. DJs can struggle to keep track of peoples' requests manually. This gave us a starting point for our app.

Process

1. Concept Development

Based on the problems we identified for both the party goers and the DJs, our proposed solution was that party goers would be able to scan a QR code around the party venue. This would allow them to join a party session and start requesting tracks.

The party session would be created by the DJ, who would also have the option to set the genre for the session.

When the user requests a track, it would be added to the DJs incoming requests list, and the DJ would be able to approve or deny the request.

2. User Stories

We created user stories so we could outline and summarise the functionality of our app. We followed the template "As a role I can capability, so that benefit".

An example of a user story we created was: "As a DJ I want to be able to share the party, so that users can join the party".

We added our user stories as issues in our team Github repository.

3. Prioritising Features

We used MoSCoW prioritisation categorties to split our user stories into Must have, Should have, Could have and Won't have categories.

Code

This was my first brief introduction to React / React Native, and it was a great opportunity to cooperate with and learn from the international students who had some experience with it.

1. Minimum Viable Product (MVP)

I focused on creating a simple track list with a request button, search function and modal confirmation pop up for our prototype.

Other group members with more experience worked on integrating the Spotify API, having a working QR scanner and working with Firebase to log the sessions created by the DJs.

2. Styling

We worked both with creating stylesheets for the different screens through creating variables in e.g. our HomeScreen.js file, and with creating a global stylesheet for the entire app which we could then export and import into the various page files.

3. Weather

I also used the OpenWeather API to add a weather widget to the home screen. We wanted users in a club to be able to see what the weather was like outside before leaving the party.

I added the API key to my JS and used the useEffect hook and fetch method to get the data from the API. I then used the data to display the temperature and weather description, based on the user's location.
