Grundfos Kollegiet

Grundfos Kollegiet

1st Semester UX/UI & Coding


Task: Devise and design a user-friendly, relevant and bold solution for a smart screen in the atrium.

Here our focus was on designing a solution for the smart screen in a student dorm atrium. We wanted to create a solution that would be relevant and useful for the students, and that would be easy to use.

We spent the first 3+ weeks on the research and prototyping stage, before working on a realisation strategy, and then used the remaining time to prepare our coded solution, report and presentation.

User and Client Research, Testing, Figma, WP

8 weeks, group project

Process

I followed the double diamond method throughout the project, and started with discovering, then defining the problem.

1. Research

Through user reseach I discovered some of the main challenges faced by potential users of the site. They faced:

  • A lack of information about the facilities
  • Wondering whether a new place is interesting enough to be worth a visit
  • Unclear directions

They also wanted to have all the information available "at a glance".

2. Design

Next I worked on developing the solution, using HMW questions to make sure I answered the users' challenges. I also created a user flow to make sure the site architecture made sense.

Some of the HMW questions I looked into included:

  • HMW let users read about others' experiences?
  • HMW encourage users to keep coming back to the site?
  • HMW give users a taste / /experience of the location?
  • HMW make the site user friendy?
  • HMW show everything in one place so users don't have to visit multiple sites?

I created a mindmap, moodboard and style tile to influence the direction of the project, and worked on lo-fi wireframes and hi-fi prototypes.

3. Testing and Iterations

I did user testing in person of the lo-fi wireframes to catch any issues at an early stage, and remote hi-fi testing to catch further usability issues. Both times I used Think Aloud testing, and feedback capture grids to gather the information. Based on testing, I made the following changes throughout the process:

  • Added an overview map on the home page showing all locations
  • Made changes to the language switch text to make it clear what langauge page the user is on
  • Added text to the scroll arrow to encourage scrolling instead of clicking.

Code

Places I enjoyed experimenting with code during the "deliver" part of the project included:

1. Mobile navigation

I added JavaScript to the close button in the mobile navigation menu to let the user return to the page they'd previously been on when tapping it.

This was a small thing, but I liked how much it improved the user experience when they were able to tap the 'x' symbol and return to where they'd previously been.

2. Adventure Options

It was interesting figuring out how to make the images have the diamond shape I'd planned for in my design.

I set up the HTML with the images in containers. The images and containers had separate classes assigned to them. I could then use CSS to rotate the container in one direction and the image in the other direction to give the effect I wanted.

3. Mapbox

I used Mapbox to show the locations of the hidden places on the website. This involved designing an overview map on the homepage and individual location-specific maps on each subpage. The primary goal was to enhance user experience by providing all necessary location information within the website, eliminating the need for users to visit external sites to find addresses.

I added the Mapbox libraries to my JS and initialised the map using my own API key. I also customised it by adding popups with the location names.
