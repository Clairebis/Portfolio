Grundfos Kollegiet

Grundfos Kollegiet

1st Semester UX/UI & Coding


Task: Devise and design a user-friendly, relevant and bold solution for a smart screen in the entrance.

Here our focus was on designing a solution for the smart screen in a student dorm entrance. We wanted to create a solution that would be relevant and useful for the students, and that would be easy to use.

We spent the first 3+ weeks on the research and prototyping stage, before working on a realisation strategy, and then used the remaining time to prepare our coded solution, report and presentation.

User and Client Research, Testing, Figma, WP

8 weeks, group project

Process

We followed the double diamond method throughout the project, and started with discovering, then defining the problem.

1. Research

Through user reseach we discovered some of the main challenges faced by potential users of the site. They faced:

  • No central place to get information about the dorms. We found people relying on Facebook groups and word of mouth for information. They felt they wasted time finding out relevant information.
  • A hard time doing laundry - they struggled to find times when the laundry machines were free.
  • A challenge wanting to be social and participate in dormitory life, but not knowing how to get information about the events.

Users would like the screen to be quick and easy to use and to be a central place to find information about the dorms.

2. Design

Next we worked on developing the solution. We used HMW questions to make sure we answered the users' issues. We also did detailed sketches and wireframes to figure out the layout of the site.

Some of the HMW questions we worked with included:

  • HMW help the users find the information they need?
  • HMW help users join social events?
  • HMW help the users improve their time management?
  • HMW visualise the facilities on the building overview?
  • HMW show everything in one place so users don't have to visit multiple sites?

Our mindmap, moodboard and style tile all helped us align, and influenced the direction of the project, particularly when it came to creating our hi-fi prototypes.

3. Testing and Iterations

We did in person user testing of our lo-fi and hi-fi prototypes using the Think Aloud method and feedback capture grids to gather the feedback. We created a list of user tasks and corresponding goals for testers to carry out. Based on testing, we made changes throughout the process. Changes we made include:

  • Adding larger clickable areas around our buttons, e.g. around our search and help icons.
  • Adding clear filter options with icons at the top of the news page so users can immediately see what they can filter by and get relevant information.
  • Making sure our button styles were consistent throughout the site.

Code

Places I enjoyed experimenting with code during the "deliver" part of the project included:

1. News Section

I added JavaScript to the close button in the mobile navigation menu to let the user return to the page they'd previously been on when tapping it.

This was a small thing, but I liked how much it improved the user experience when they were able to tap the 'x' symbol and return to where they'd previously been.

2. Events Section

It was interesting figuring out how to make the images have the diamond shape I'd planned for in my design.

I set up the HTML with the images in containers. The images and containers had separate classes assigned to them. I could then use CSS to rotate the container in one direction and the image in the other direction to give the effect I wanted.
